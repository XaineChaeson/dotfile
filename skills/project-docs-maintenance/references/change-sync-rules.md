# Change Sync Rules

Use these as project-specific rules. Fill in real paths.

- If a core interface changes, update the matching interface doc.
- If interface signatures change, update API overview table entries (capability/entrypoint only).
- If a module boundary changes, update `docs/dev/architecture.md` and module docs.
- If config defaults change, update CONFIG and examples.
- If a new adapter/implementation is added, update overview table.
- If behavior changes, update devlog or changelog.
