# Docs Structure Reference

Use this to document the canonical doc layout and ownership for a project.

## Directory layout

```text
README.md
docs/
  README.md
  project-status.md
  prd/
  architecture/
  manual/
  dev/
  scenarios/
  ops/
  reference/
  archive/
```

Remove layers that do not apply, but keep architecture, manual, and dev separate for substantial systems.
