# Adapter Delta (Instance Deviations)

Use this to document differences between a concrete adapter and the interface contract.

## Adapter
- Name and location.
- Interface contract reference.

## Deviations
- Unsupported functions or parameters.
- Default values that differ from the contract.
- Edge cases or platform-specific limits.

## Config differences
- Required config that is optional in the contract.
- Overrides to defaults.

## Runtime requirements
- RPC/chain requirements, addresses, or other dependencies.

## Performance and limits
- Rate limits, timeouts, size bounds.

## Errors and behaviors
- Special error cases or non-standard return behavior.
